## 💻 项目经历

### 1. "Shadow Gate" 恶意加载器与 CTF 挑战设计

* **项目描述**：独立设计并部署一个面向高阶 CTF 竞赛的二进制安全（Pwn/Reverse）挑战靶场。


* **技术要点**：
* **控制流劫持**：编写自定义**恶意加载器（Loader）**，通过修改 ELF 文件的初始化段（如 `.init_array`）或劫持控制流，成功在 `main` 函数执行前的启动序列（Startup Sequence）中注入对抗逻辑，显著提升逆向分析难度。


* **靶场弹性架构**：编写 **Dockerfile** 和 `docker-compose.yml` 配置文件，一键编排并部署高度隔离的 PHP Web 后端与二进制漏洞验证容器，具备强抗炸、防非预期利用的安全沙箱特性。





### 2. 基于虚拟机栈的混淆指令集系统 (baby_vm)

* **项目描述**：使用 **C 语言** 独立编写的黑盒虚拟机，设计并执行一套自定义的私有指令集，用于模拟软件保护中的 VM 混淆对抗。


* **技术要点**：
* **取指译码仿真**：完整模拟了 CPU 核心的**取指令-译码-执行**状态机流程，实现了一套非标准的 VCPU 运行环境。


* **逆向对抗设计**：采用虚拟栈（Virtual Stack）架构进行多数据隔离，将真实逻辑剥离至自定义字节码中，用于深入研究 VM-based 混淆控制流平坦化以及探索虚拟机逃逸（VM Escape）的物理边界。





### 3. 端到端加密分布式文件系统 (ShareLock)

* **项目描述**：参考 UC Berkeley CS161 安全框架，构建在完全不可信（恶意主动攻击者控制）的远程服务器之上的端到端加密存储系统。


* **技术要点**：
* **工程稳健性**：全流程采用 **TDD（测试驱动开发）** 模式，前置设计对抗边界单测，确保复杂密码学状态切换时代码的绝对质量。


* **密码学原语应用（E2EE）**：设计完备的客户端端到端加密体系。利用 **Argon2** 算法对抗密码暴力破解，结合非对称/对称混合加密确保密钥、文件名及共享依赖图（Sharing Graph）在本地设备外零泄露。


* **密文防篡改（Encrypt-then-MAC）**：针对大文件设计了无需重新加密全量数据的**安全追加（Secure Append）机制**；密文在持久化写入嵌入式 **BadgerDB** 前强制进行 `HMAC-SHA512` 认证，彻底阻断重放攻击与中间人密文篡改。


* **动态撤销与控制**：利用动态加密邀请指针（Invitation Pointer）实现细粒度共享。设计动态密钥轮换（Key Rotation）机制，在触发 `revoke` 权限撤销时，立即使被撤销用户的历史密钥失效，并透明更新剩余合法节点的密钥拓扑。





### 4. xv6 现代操作系统内核特性扩展 (MIT 6.S081)

* **项目描述**：深入仿 Unix 的进程隔离内核源码，研究并增强操作系统的虚拟内存、系统调用与底层内存对抗特性。


* **技术要点**：
* **页表级隔离**：重新规划内核页表（Page Tables）布局，实现用户态与内核态的高效、安全数据边界拷贝，深度掌握内核权限提升与系统调用拦截的底层逻辑。


* **并发安全 COW 实现**：拦截 `fork()` 时的物理内存全量复制，通过引入**引用计数（Reference Counting）**和内核**页面保护中断**（捕捉写页错误并动态按需分配新物理页）实现写时复制机制。




* **项目成果**：代码完全通过 `pgtbltest` 和 `cowtest` 严苛的并发与压力测试，大幅度降低了系统的内存开销。
